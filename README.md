# HospitalOps

HospitalOps is an object-oriented hospital management system that integrates patient queue handling with inventory management. It enables prioritized patient processing, doctor consultations, and automatic deduction of medical supplies used during treatment, ensuring efficient and realistic hospital operations.

## Features
Priority-based patient queue (Emergency, Urgent, Non-Urgent)
Doctor consultation management
Inventory tracking for medicines, equipment, and supplies
Automatic deduction of items after patient treatment
Role-based user system (Admin, Nurse, Doctor)

## OOP Concepts Applied
Encapsulation – Secure handling of patient and inventory data
Inheritance – User roles extend from a base User class
Polymorphism – Different behaviors depending on user roles
Composition – TreatmentRecord contains items used during treatment

## System Flow
Patient is registered and added to the queue
Queue is prioritized based on condition level
Doctor conducts consultation
TreatmentRecord is created after consultation
Inventory is automatically updated based on items used

## Technologies Used
Object-Oriented Programming (OOP)
Language: Java
IDE: Visual Studio
